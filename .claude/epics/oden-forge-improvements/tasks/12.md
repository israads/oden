# Task: Extended Configuration Wizard

## Description
Extend the existing `/oden:init` wizard to configure new productivity features during project setup. Add configuration options for bug resolution preferences, pipeline stages, export formats, dashboard settings, and team collaboration features.

## Functional Requirements
- **Feature Selection**: Configure which new productivity features to enable
- **Bug Resolution Preferences**: Choose auto-fix vs manual approval settings
- **Pipeline Configuration**: Set default stages and specialized agents
- **Export Preferences**: Configure client branding and default formats
- **Dashboard Settings**: Auto-start preferences and monitoring options
- **Team Configuration**: Shared settings via `.oden-config.json`

## Technical Implementation

### Files to Modify:
- `.claude/commands/oden/init.md` - Enhanced wizard definition
- `lib/commands/init.js` - Extended configuration logic
- `lib/init/feature-config.js` - New feature configuration system
- `lib/init/team-config.js` - Team configuration sharing logic
- `templates/.oden-config.json` - Configuration file template

### Architecture:
1. **Wizard Extension**: Add new configuration steps to existing flow
2. **Feature Detection**: Determine which features are available/relevant
3. **Configuration Generation**: Create comprehensive `.oden-config.json`
4. **Team Integration**: Support git-committable team configurations
5. **Validation**: Ensure configuration consistency and validity

## Configuration Wizard Flow

### New Steps Addition:
1. **Feature Selection**: "Which productivity features do you want to enable?"
2. **Bug Resolution**: "How should bug fixes be handled?" (auto/manual/prompt)
3. **Agent Pipeline**: "Which code quality agents should run?" (multi-select)
4. **Export Setup**: "Configure documentation export preferences"
5. **Team Settings**: "Is this a team project?" (individual/team configuration)

### User Experience:
- [ ] Clear explanations of what each feature does
- [ ] Examples of how settings affect workflow
- [ ] Option to skip advanced configuration
- [ ] Preview of generated configuration file
- [ ] Validation feedback for configuration conflicts

## Risks
- **Medium**: Configuration complexity might overwhelm new users
- **Medium**: Team configuration conflicts could cause confusion
- **Low**: Integration with existing projects might be challenging

## Mitigation Strategies
- Progressive disclosure: basic setup first, advanced options later
- Comprehensive documentation and examples
- Clear conflict resolution UI and workflows
- Extensive testing with various project types and team structures
